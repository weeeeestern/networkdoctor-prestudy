### **Kiali: 서비스 간의 지도**

- **토폴로지 그래프:** 서비스 A가 B를 호출할 때 트래픽이 얼마나 흐르는지, 에러가 발생하는지 화살표 방향과 선의 굵기(애니메이션)로 실시간 시각화 해 줍니다.
- **Istio 설정 검증:** Istio의 설정 파일(VirtualService, DestinationRule 등)이 올바르게 작성되었는지 검증하고 편집하는 기능을 제공합니다.
- **Kiali AI Bot (2026 신기능):** 최근 Kiali에 도입된 AI 챗봇은 "최근 5분간 에러율이 높은 서비스는 뭐야?" 또는 "특정 서비스의 트래픽을 차단해줘" 같은 자연어 명령을 처리하거나 복잡한 메시 구조 내에서 길찾기를 도와줍니다.

### **Grafana: 지표의 "히스토리(History)"**

- **대시보드 중심:** CPU 사용량, 메모리 점유율, 응답 속도(Latency) 등의 데이터를 꺾은선 그래프나 게이지로 보여주는 데 특화되어 있습니다.
- **다양한 데이터 소스:** Prometheus뿐만 아니라 InfluxDB, Elasticsearch, CloudWatch 등 거의 모든 데이터 소스를 연결해 통합 대시보드를 구성할 수 있습니다.
- **정밀 분석:** 특정 시점에 장애가 발생했을 때, 과거의 지표와 비교하여 어떤 자원이 부족했는지 심층 분석하는 데 유리합니다.

- **Kiali**를 쓸 때:
    - "내 서비스들이 서로 어떻게 통신하고 있지?" (전체 구조 파악)
    - "특정 구간에서 트래픽 병목이나 에러가 나는데 어디서 시작된 거지?"
    - "Istio 설정을 GUI에서 편하게 관리하고 싶다."
- **Grafana**를 쓸 때:
    - "서버 CPU나 메모리 점유율 추이를 보고 싶다."
    - "서비스의 골든 시그널(에러율, 지연시간 등)을 한눈에 보는 고정된 대시보드가 필요하다."
    - "특정 지표가 임계치를 넘으면 슬랙 알림을 받고 싶다."

> **💡 팁:** 실제 운영 환경에서는 이 둘을 **함께** 사용합니다. Kiali에서 전체적인 트래픽 흐름과 장애 지점을 찾고, 해당 서비스의 상세한 리소스 지표는 연결된 Grafana 링크를 클릭해 정밀하게 확인하는 방식이 가장 일반적입니다.
